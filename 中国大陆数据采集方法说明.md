# 中国大陆加密货币数据采集方法说明文档

本文档总结了本项目中用于在中国大陆网络环境下获取加密货币历史数据和实时数据的方法。这些方法经过实践验证，能够有效解决网络受限、API访问困难等问题。

## 核心策略概述

本项目采用 **"三层数据获取策略"**，按优先级保障数据的可用性和准确性：

1.  **第一层：国内可用聚合API** (稳定性高，适合历史数据)
2.  **第二层：国内网页抓取** (无门槛，适合实时价格)
3.  **第三层：直连交易所/全球API** (需特定网络配置或镜像)

---

## 方法一：国内可用聚合API (推荐)

这些服务通常提供对中国大陆友好的API端点，适合获取高质量的历史K线数据。

### 1. 数据源列表

| 数据源 | 接口地址 | 类型 | 说明 |
| :--- | :--- | :--- | :--- |
| **iTick** | `https://api.itick.com` | 聚合器 | 提供历史和实时数据，需API Key |
| **沧海数据 (Tsanghi)** | `https://tsanghi.com/api/fin` | 聚合器 | 专注于金融数据，国内访问稳定 |
| **Infoway** | `https://api.infoway.com` | 聚合器 | 提供多维度的加密数据 |

### 2. 实现示例

使用 `requests` 库访问，建议配置 User-Agent 模拟浏览器。

```python
import requests
import datetime

def fetch_history_from_tsanghi(symbol, days, api_token):
    """
    从沧海数据获取历史数据
    :param symbol: 交易对，如 'BTC/USDT'
    :param days: 获取天数
    :param api_token: API Token
    """
    base, quote = symbol.split('/')
    end_date = datetime.datetime.now().strftime('%Y-%m-%d')
    start_date = (datetime.datetime.now() - datetime.timedelta(days=days)).strftime('%Y-%m-%d')
    
    url = "https://tsanghi.com/api/fin/crypto/daily"
    params = {
        'token': api_token,
        'ticker': base,
        'currency': quote,
        'start_date': start_date,
        'end_date': end_date
    }
    
    headers = {'User-Agent': 'Mozilla/5.0 ...'}
    
    try:
        resp = requests.get(url, params=params, headers=headers, timeout=20)
        if resp.status_code == 200:
            return resp.json()
    except Exception as e:
        print(f"Error: {e}")
    return None
```

## 方法二：网页抓取 (Web Scraping)

通过抓取国内可访问的财经网站页面，利用正则表达式提取价格。这种方法不需要API Key，适合获取最新的实时价格作为验证或后备。

### 1. 目标网站

*   **金色财经 (Jinse)**: `https://m.jinse.cn/coin/search?coin={slug}`
*   **非小号 (Feixiaohao)**: `https://www.feixiaohao.com/currencies/{slug}/`
*   **Investing.com (CN)**: `https://cn.investing.com/crypto/{slug}/{slug}-usd`
*   **新浪区块链**: `https://finance.sina.com.cn/blockchain/hq.shtml`
*   **528btc**: `https://www.528btc.com/coin/{slug}/`

### 2. 关键技术点

*   **Regex 提取**: 由于页面结构可能变化，使用多个正则表达式匹配价格模式是最佳实践。
*   **Symbol 映射**: 需要维护一个 `Symbol -> Slug` 的映射表 (例如 `BTC` -> `bitcoin`)。

### 3. 实现示例

```python
import requests
import re

def extract_usd_price(text):
    """使用多种正则模式提取价格"""
    patterns = [
        r'"price"\s*:\s*([0-9]+(?:\.[0-9]+)?)',
        r'(?:\$|USD)[^0-9]*([0-9]{1,3}(?:,[0-9]{3})*(?:\.[0-9]+)?)',
        r'data-last\s*=\s*"([0-9]+(?:\.[0-9]+)?)"'
    ]
    for p in patterns:
        m = re.search(p, text, flags=re.IGNORECASE)
        if m:
            return float(m.group(1).replace(',', ''))
    return 0.0

def fetch_jinse_price(symbol_slug):
    url = f'https://m.jinse.cn/coin/search?coin={symbol_slug}&space=tencent'
    headers = {
        'User-Agent': 'Mozilla/5.0 ...',
        'Accept-Language': 'zh-CN,zh;q=0.9'
    }
    try:
        resp = requests.get(url, headers=headers, timeout=8)
        if resp.status_code == 200:
            return extract_usd_price(resp.text)
    except:
        pass
    return None
```

## 方法三：直连交易所优化

虽然部分交易所API受限，但通过特定配置仍可能建立连接，或利用部分交易所未完全屏蔽的端点。

### 1. 连接优化技巧

*   **Session配置**: 使用 `requests.Session()` 复用连接。
*   **SSL验证**: 在受信任的内部网络或特定网络环境下，禁用 SSL 验证 (`verify=False`) 可能解决部分握手失败问题（**注意安全风险**）。
*   **Headers伪装**: 必须伪装成标准浏览器。
*   **备用域名**: 尝试交易所的不同API域名 (如 `api.huobi.pro` vs `api-aws.huobi.pro`，视网络情况而定)。

### 2. 配置示例

```python
import requests
import urllib3

# 禁用SSL警告
urllib3.disable_warnings(urllib3.exceptions.InsecureRequestWarning)

def create_optimized_session():
    session = requests.Session()
    # 连接池优化
    adapter = requests.adapters.HTTPAdapter(pool_connections=10, pool_maxsize=100)
    session.mount('https://', adapter)
    session.verify = False # 仅在必要时使用
    session.headers.update({
        'User-Agent': 'Mozilla/5.0 ...',
        'Accept-Language': 'zh-CN,zh;q=0.9',
        'Connection': 'keep-alive'
    })
    return session
```

## 数据共识机制 (Consensus Mechanism)

为了应对单一数据源可能出现的异常（如爬虫抓取错误或API延迟），建议实现**共识机制**。

1.  **采集**: 并行从上述多个来源获取价格。
2.  **过滤**: 剔除异常值 (如偏离中位数 > 15% 的数据)。
3.  **计算**: 计算剩余有效数据的平均值或中位数作为最终价格。
4.  **权重**: 可给予 API 类数据源更高权重，爬虫类数据源较低权重。

```python
def calculate_consensus_price(prices):
    if not prices: return 0.0
    # 简单的中位数过滤逻辑
    prices.sort()
    mid = len(prices) // 2
    median = prices[mid]
    
    # 只保留偏差在15%以内的价格
    valid_prices = [p for p in prices if abs(p - median)/median <= 0.15]
    
    if not valid_prices: return median
    return sum(valid_prices) / len(valid_prices)
```

对于需要在国内环境稳定运行的项目，建议：
1.  **首选**申请并配置 `Tsanghi` 或 `iTick` 等聚合API Key。

3.  **保留** 全球API (CoinGecko/Binance) 作为备用，并结合代理设置使用。
